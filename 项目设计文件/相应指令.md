## 一、后端项目初始化

### 1.1 环境要求

| 工具 | 版本 |
|:----:|:----:|
| Node.js | ≥ 22 |
| npm | ≥ 10 |
| PostgreSQL | 15+ |
| Redis | 7+ |
| Docker | 可选（用于启动数据库） |

### 1.2 初始化步骤

#### ① 安装 NestJS CLI（如未安装）

```bash
npm install -g @nestjs/cli
```

#### ② 创建 NestJS 项目

```bash
cd D:\old\项目\工程实训（二）
nest new backend --skip-git --package-manager npm --strict --language TypeScript
```

#### ③ 安装核心依赖

```bash
cd backend

# 生产依赖
npm install @nestjs/config @nestjs/jwt @nestjs/passport @nestjs/platform-socket.io @nestjs/websockets @nestjs/bullmq @prisma/client prisma passport passport-jwt bcrypt bullmq ioredis socket.io class-validator class-transformer pdf-parse mammoth multer openai rxjs

# 开发依赖
npm install -D @types/bcrypt @types/multer @types/passport-jwt @types/pdf-parse ts-node nodemon
```

> **注意：** 如果安装的 Prisma 版本为 7.x，需要降级到 5.x 以获得传统模式支持：
>
> ```bash
> npm install prisma@5 @prisma/client@5
> ```

#### ④ 初始化 Prisma

```bash
npx prisma init
```

#### ⑤ 配置环境变量

编辑 `.env` 文件，配置数据库连接等信息：

```env
# Database
DATABASE_URL="postgresql://app:app123456@localhost:5432/career_copilot"

# Redis
REDIS_URL="redis://localhost:6379/0"

# JWT
JWT_SECRET="your-jwt-secret-key-change-in-production"
JWT_ACCESS_EXPIRES="15m"
JWT_REFRESH_EXPIRES="7d"

# LLM 配置
LLM_PROVIDER=                  # 可选，显式指定 openai / dashscope / deepseek，留空则自动检测
OPENAI_API_KEY="sk-xxx"
OPENAI_BASE_URL="https://api.openai.com/v1"
OPENAI_MODEL="gpt-4o-mini"     # 可选，默认 gpt-4o-mini
DASHSCOPE_API_KEY="sk-xxx"
DASHSCOPE_BASE_URL="https://dashscope.aliyuncs.com/compatible-mode/v1"
DASHSCOPE_MODEL="qwen-turbo"   # 可选，默认 qwen-turbo
DEEPSEEK_API_KEY="sk-xxx"
DEEPSEEK_BASE_URL="https://api.deepseek.com/v1"
DEEPSEEK_MODEL="deepseek-chat" # 可选，默认 deepseek-chat

# 文件上传
UPLOAD_DIR="./uploads"
MAX_FILE_SIZE=10485760

# 服务器
PORT=3002
```

#### ⑥ 配置 Prisma Schema

`prisma/schema.prisma` — 定义数据模型（User、Resume、Interview、InterviewMessage、CareerPlan），详见 `项目设计文件/database_design.md`。

#### ⑦ 生成 Prisma Client

```bash
npx prisma generate
```

#### ⑧ 验证编译

```bash
npx nest build
```

编译成功应无任何错误输出。

#### ⑨ 启动数据库（Docker）

> ⚠️ **国内网络说明：** Docker Hub 在国内访问不稳定。推荐**关闭梯子**，使用国内镜像加速。

**首次使用需先配置 Docker 国内镜像加速（只需配一次）：**

在 `C:\Users\你的用户名\.docker\daemon.json` 中写入以下内容：

```json
{
  "registry-mirrors": [
    "https://docker.m.daocloud.io",
    "https://dockerproxy.com",
    "https://docker.nju.edu.cn"
  ]
}
```

配置完后**重启 Docker Desktop**（系统托盘右键 → Restart），然后启动容器：

```bash
# 启动 PostgreSQL + Redis（后台运行）
docker compose up -d

# 查看容器是否正常运行
docker compose ps

# 查看启动日志
docker compose logs

# 停止并删除容器
docker compose down

# 彻底清理（删除容器 + 数据卷）
docker compose down -v
```

> 📌 **镜像说明：** `docker-compose.yml` 中已配置国内镜像地址 `docker.m.daocloud.io/library/...`，如有梯子想用官方源，把 `docker.m.daocloud.io/library/` 前缀去掉即可。

#### ⑩ 验证数据库连接

确认 PostgreSQL 和 Redis 已就绪：

```bash
# 检查 PostgreSQL 是否可连接
docker compose exec postgres pg_isready -U app

# 检查 Redis 是否可连接
docker compose exec redis redis-cli ping
# 应返回 PONG

# 或使用 npx 验证 Prisma 连接
npx prisma db push --accept-data-loss
```

#### ⑩ 运行数据库迁移

```bash
npx prisma migrate dev --name init
```

#### ⑪ 启动开发服务器

若报错端口占用，可使用下面表格中的指令释放端口

```bash
npm run start:dev
```

服务运行在 <http://localhost:3002>

### 1.4 Docker 国内镜像加速配置

Docker Hub 在国内访问不稳定，推荐配置国内镜像加速（替代开梯子）：

**配置步骤：**

| 步骤 | 操作 | 说明 |
|:----:|------|------|
| 1 | **关闭梯子（VPN/代理）** | 梯子和国内镜像源会有冲突 |
| 2 | 编辑 `C:\Users\你的用户名\.docker\daemon.json` | 写入镜像加速源地址 |
| 3 | **重启 Docker Desktop** | 系统托盘右键 → Restart |
| 4 | `docker compose up -d` | 从国内镜像拉取并启动 |
| 5 | `docker compose ps` | 确认两个容器都是 `Up` 状态 |

**daemon.json 完整配置参考：**

```json
{
  "registry-mirrors": [
    "https://docker.m.daocloud.io",
    "https://dockerproxy.com",
    "https://docker.nju.edu.cn",
    "https://mirror.ccs.tencentyun.com"
  ]
}
```

> **如果拉取镜像时卡住或报 `EOF` 错误：** 一般是梯子没关干净或者 Docker Desktop 没重启。先关梯子，重启 Docker Desktop，再试一次。

### [x] T-011 — 面试 Prompt 工程 + AI 面试引擎（陶宏阳）✅

> **已实现功能：** 面试官 System Prompt 构建器 + AI 面试引擎核心（出题→评估→追问/下一题/结束），集成到 InterviewService。

**文件结构：**

```
src/ai/prompts/
└── interview.system.ts           # 面试官 System Prompt 构建器

src/interview/
├── ai-interview.service.ts       # AI 面试引擎核心
├── interview.service.ts          # 集成 AI 面试（create 自动出题，submitAnswer AI 评估）
└── interview.module.ts           # 注册 AiInterviewService + 引入 AiModule
```

**System Prompt 构建器 (`interview.system.ts`)：**

根据职位、难度、简历上下文动态生成面试官提示词，定义三种输出格式：

| 类型 | 触发场景 | 说明 |
|:----:|----------|------|
| `question` | 出题 | `{ type, content, questionType }` |
| `evaluation` | 评估回答后 | `{ score, feedback, nextAction }` |
| `complete` | 结束面试 | `{ summary }`（含在 evaluation 内） |

**`nextAction` 三种状态：**

| 动作 | 含义 | 后续处理 |
|:----:|------|----------|
| `followUp` | 继续追问 | 保存追问内容，不增加 questionCount |
| `nextQuestion` | 进入下一题 | 保存新题，questionCount++ |
| `complete` | 结束面试 | 更新 status=completed，记录分数 |

**AI 面试引擎 (`ai-interview.service.ts`)：**

```typescript
@Injectable()
export class AiInterviewService {
  // 生成第一道题
  generateFirstQuestion(context: InterviewContext): Promise<FirstQuestionResult>

  // 评估回答 + 决定后续动作
  evaluateAndContinue(context, userAnswer, history): Promise<EvaluationResult>
}
```

**面试对话流程：**

```
POST /interviews
  │
  ├─ 创建面试记录
  ├─ 读取简历上下文（如有 resumeId）
  ├─ 调用 AiInterviewService.generateFirstQuestion()
  └─ 保存首题 → 返回 { interview, firstQuestion }
       │
       ▼ 用户答题
POST /interviews/:id/answer  { content }
  │
  ├─ 保存用户回答
  ├─ 构建完整对话上下文
  ├─ 调用 AiInterviewService.evaluateAndContinue()
  ├─ 根据 nextAction 处理后续
  │   ├─ followUp → 保存追问，返回下一轮
  │   ├─ nextQuestion → 保存新题，返回下一轮
  │   └─ complete → 完成面试，保存总分
  └─ 返回 { evaluation, nextQuestion?, isComplete }
       │
       ▼ 循环...
POST /interviews/:id/answer → ... → isComplete=true
```

POST /api/auth/login  →  复制 accessToken 到 @token=

# 2. 创建面试（自动生成首题）
POST /api/interviews  { "targetPosition": "前端开发工程师", "difficulty": "mid" }
  → 返回 firstQuestion.content（AI 出的第一道题）

# 3. 提交回答（AI 评估）
POST /api/interviews/:id/answer  { "content": "我的回答..." }
  → 返回 { score, feedback, nextQuestion?, isComplete? }

# 4. 循环答题直到 isComplete=true

# 5. 生成面试报告
POST /api/interviews/:id/feedback
  → 返回多维度评分报告
```

**首次使用：**

| 步骤 | 操作 |
|:----:|------|
| 1 | 安装 VS Code 插件：**REST Client**（`humao.rest-client`） |
| 2 | 打开 `backend/test-api.http` |
| 3 | 找到 **2.2 登录**，点击上方的「Send Request」 |
| 4 | 登录返回的 `accessToken` 粘贴到文件顶部 `@token =` 后面 |
| 5 | 找到 **5.1 创建面试** 等接口，点「Send Request」测试 |

**面试模块测试流程（完整示例）：**

```
① 5.1 创建面试 → 返回 interviewId + firstQuestion
   ↓
② 将返回的 interviewId 填入 @interviewId =
   ↓
③ 5.5 提交回答 → 返回 score + nextQuestion
   ↓

### 方式二：Swagger UI（推荐前端/全栈用）

> 在浏览器中可视化调用接口，适合调试和演示。

**首次使用：**

| 步骤 | 操作 |
|:----:|------|
| 1 | 启动后端：`npm run start:dev` |
| 2 | 打开浏览器访问 <http://localhost:3002/api-docs> |
| 3 | 展开 **POST /api/auth/login** → 点「Try it out」→ 填入邮箱密码 → 点「Execute」|
| 4 | 在返回的 Response body 中复制 `accessToken` 的值 |
| 5 | 点页面右上角 **Authorize** 按钮 |
| 6 | 输入 `Bearer <粘贴的accessToken>` → 点 **Authorize** → 点 **Close** |
| 7 | 展开其他接口 → 点「Try it out」→ 填写参数 → 点「Execute」测试 |

**Authorize 注意事项：**

- 格式必须是 `Bearer eyJh...`（注意 **Bearer 后面有空格**）
- 不要只粘贴 token，要加 `Bearer` 前缀
- token 有效期 15 分钟，过期需重新登录并重新 Authorize
- ✅ **刷新页面后 token 会自动保留**（已配置 `persistAuthorization: true`），无需重复输入
- 如果刷新后 Authorize 还是空白，尝试**清除浏览器缓存**后再试一次

**面试模块测试流程（Swagger）：**

```
展开 POST /api/interviews
  → Try it out
  → 填写 { "targetPosition": "前端开发工程师", "difficulty": "mid" }
  → Execute → 返回 interviewId + firstQuestion

展开 POST /api/interviews/{id}/answer
  → 把 interviewId 填入 id 参数
  → Body: { "content": "你的回答" }
  → Execute → 返回 score + nextQuestion

→ 重复提交回答...
→ 面试完成后调 POST /api/interviews/{id}/feedback 生成报告
```

**测试流程：**

```
① POST /api/auth/login → 获取 accessToken
   ↓
② POST /api/interviews → 创建面试，获取 interviewId
   ↓
③ WebSocket 连接 ws://...?token=accessToken

### [x] T-012 — WebSocket 面试网关（陶宏阳）✅

> **已实现功能：** 实时流式 AI 模拟面试，支持 JWT 认证、消息持久化、模拟流式输出。

**文件结构：**

```
src/interview/
├── interview.gateway.ts       # WebSocket 网关（JWT 认证 + user_answer 处理）
└── interview.module.ts        # 注册 Gateway + 导入 JwtModule
```

**WebSocket 连接地址：** `ws://localhost:3002/ws/interview?token=<JWT>`

**消息协议：**

| 方向 | 类型 | 说明 |
|------|------|------|
| 客户端 → 服务端 | `user_answer` | `{ interviewId, content }` 提交回答 |
| 服务端 → 客户端 | `ai_message_chunk` | `{ messageId, chunk }` 流式片段 |
| 服务端 → 客户端 | `ai_message_done` | `{ messageId, fullContent, feedback, score, strengths, weaknesses, nextAction, followUpContent?, nextQuestion? }` |
| 服务端 → 客户端 | `error` | `{ code, message }` 错误消息 |

**`nextAction` 三种状态：**

| 动作 | 含义 | 服务端处理 |
|:----:|------|----------|
| `followUp` | 继续追问 | 保存追问，questionCount 不变 |
| `nextQuestion` | 进入下一题 | 保存新题，questionCount++ |
| `complete` | 结束面试 | 更新 status=completed，记录分数 |

**网关处理流程：**

```
客户端连接 (ws://...?token=xxx)
  ├─ JWT 验证 → 失败断开并返回 error
  └─ 成功 → 记录 userId
       │
客户端发送 user_answer { interviewId, content }
  ├─ 校验参数 + 面试归属
  ├─ 保存用户回答到数据库
  ├─ 获取对话历史 + 简历上下文
  ├─ 调用 AiInterviewService.evaluateAndContinue()
  ├─ 保存 AI 回复到数据库
  ├─ 更新面试状态（questionCount++ / 完成）
  ├─ 模拟流式输出 ai_message_chunk（按句子分割）
  └─ 发送 ai_message_done（含评分 + 下一动作）
```

**测试方法：**

1. **浏览器控制台：** 打开 F12 → Console，粘贴以下代码：

```javascript

ws.onmessage = (e) => console.log('收到:', JSON.parse(e.data));
ws.onopen = () => {
  console.log('已连接');
  ws.send(JSON.stringify({
    interviewId: '你的面试ID',
    content: '闭包是指有权访问另一个函数作用域中变量的函数'
  }));
};
```

1. **wscat 工具：**

```bash
