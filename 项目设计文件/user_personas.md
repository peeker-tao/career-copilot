# Career-Copilot 用户示例文档

> 版本：v1.0 | 日期：2026-06-13

---

## 一、概述

本文档定义了 Career-Copilot 平台的目标用户画像（User Personas），通过对典型用户群体的特征、目标、痛点和行为模式进行建模，指导产品设计与功能决策。每个画像基于真实调研数据（大学生求职调研、职场新人访谈）提炼而成。

---

## 二、用户画像总览

| 编号 | 画像名称 | 用户群体 | 核心需求 | 技术敏感度 |
|:----:|----------|----------|----------|:----------:|
| P-01 | 迷茫的大三学生 | 本科大三、零实习经验 | 了解岗位要求、积累面试经验 | 中 |

### P-01：迷茫的大三学生

> *"马上要大四了，我连面试长什么样都不知道……"*

| 字段 | 内容 |
|------|------|
| **姓名** | 张小明 |
| **年龄** | 21 岁 |
| **学校/专业** | 武汉某双非一本，软件工程专业 |
| **年级** | 大三（即将升大四） |
| **技术能力** | 学过 Java、Python 基础，做过课程设计，没有什么拿得出手的项目 |
| **求职状态** | 对未来的职业方向感到迷茫，不确定自己适合做前端、后端还是测试 |
| **面试经验** | 0 次面试经验，不知道面试流程是什么样的 |
| **简历情况** | 没有正式简历，只有一份课程项目清单 |

#### 🎯 目标

- 了解软件行业各岗位（前端/后端/算法/测试）到底是做什么的
- 体验一次完整的面试流程，消除对面试的恐惧
- 得到一个清晰的技能学习和提升方向

#### 😟 痛点

1. **信息差** — 不知道不同岗位的真实工作内容和要求
2. **缺乏自信** — 觉得自己什么都不会，不敢投简历
3. **无从下手** — 不知道应该先学什么、学到什么程度
4. **简历空白** — 没有实习经历，简历拿不出手

#### 📱 使用场景

- 周末晚上在宿舍，打开 Career-Copilot 选择"后端开发"岗位进行一次轻松模式的模拟面试
- 面试后看到评分和建议，发现自己对数据库问题完全不会回答
- 转而使用职业规划功能，输入"目标：后端开发"系统生成了一份包含 Java、MySQL、Spring Boot 的学习路线

#### ✅ 成功标准

- 能够自信地说出"我知道面试是什么样的了"
- 根据学习路线规划出大三暑假和大四上的学习计划
- 有了明确的目标岗位（后端开发），不再迷茫

### P-02：冲刺的应届毕业生

> *"秋招已经开始一个多月了，我面了 5 家全挂了，问题到底出在哪？"*

| 字段 | 内容 |
|------|------|
| **姓名** | 李思雨 |
| **年龄** | 22 岁 |
| **学校/专业** | 武汉某 211 高校，计算机科学与技术专业 |
| **年级** | 大四（正在参加秋招） |
| **技术能力** | 成绩不错，有 1 段中小厂实习经历，LeetCode 刷了 200+ |
| **求职状态** | 海投了 30+ 家公司，收到 8 个笔试，最终面试 5 家全部一轮游 |
| **面试经验** | 5 次真人面试经历，但每次都很紧张、表达混乱 |
| **简历情况** | 有实习经历，项目经验写了 2 个 |

#### 🎯 目标

- 找出面试屡屡失败的根本原因
- 针对性地提升面试表达能力和答题技巧
- 在接下来的面试中通过至少 2 家公司的最终面

#### 😟 痛点

1. **表现失常** — 明明会的问题，面试时一紧张就说不清楚
2. **缺乏反馈** — 面试被拒后只知道"不合适"，不知道具体哪里不好
3. **表达无条理** — 回答项目经历时像记流水账，缺乏 STAR 逻辑
4. **压力面不适应** — 遇到面试官追问就慌了

#### 📱 使用场景

- 每天花 30 分钟在平台上进行针对性的"产品经理/后端开发"岗位面试，选择**困难模式**
- 使用"语音面试"功能模拟真实的线上面试环境
- 仔细阅读每轮 AI 给出的反馈，尤其是关于"表达逻辑"和"回答深度"的点评
- 查看面试报告中的"改进建议"，针对薄弱点反复练习

#### ✅ 成功标准

- 面试报告评分从第一次的 58 分提升到第 10 次的 82 分
- 在后续的真实面试中通过了一家互联网中厂的二面
- 能够有条理地使用 STAR 法则介绍项目经历

### P-03：跨专业求职者

> *"本科学的机械，自学了 Python 和机器学习，想转行算法工程师，但简历投出去都没回音……"*

| 字段 | 内容 |
|------|------|
| **姓名** | 王浩然 |
| **年龄** | 23 岁 |
| **学校/专业** | 华中某 985 高校，机械工程专业（硕士在读） |
| **技术能力** | 自学 Python、机器学习基础、Kaggle 打过 2 场比赛 |
| **求职状态** | 投了 15 家算法岗，仅收到 2 个笔试，0 个面试 |
| **面试经验** | 无算法岗位面试经验 |
| **简历情况** | 有机械相关的项目和论文，但算法方向项目薄弱 |

#### 🎯 目标

- 了解算法工程师岗位的真实技能要求
- 对比自身技能差距，制定系统的补强计划
- 积累算法面试经验

#### 😟 痛点

1. **专业不对口** — 简历筛选阶段就因为专业被刷
2. **技能树不完整** — 不知道自己还缺什么技能
3. **项目经验不足** — 缺乏有分量的算法项目来证明能力
4. **不了解行业要求** — 企业到底需要什么层次的算法能力？

#### 📱 使用场景

- 上传本科学的课程项目和 Kaggle 比赛简历进行解析
- 使用职业规划功能，选择"目标：算法工程师"，系统分析出他的技能差距：
  - ✅ 已掌握：Python、机器学习基础、Pandas
  - ❌ 待学习：深度学习框架（PyTorch/TensorFlow）、数据结构与算法（手撕代码）、SQL、部署相关（Docker）
- 根据差距，系统推荐了 Andrew Ng 的深度学习课程和一个推荐系统项目实战
- 每周进行 2-3 次算法岗位模拟面试

#### ✅ 成功标准

- 明确自己的技能差距并有了清晰的补强计划
- 经过 2 个月学习后，能够通过模拟面试达到 70+ 分
- 最终拿到至少 1 个算法相关的实习面试机会

### P-04：职场跳槽者

> *"在现在的公司待了两年，技术成长太慢了，想看看外面的机会怎么样。"*

| 字段 | 内容 |
|------|------|
| **姓名** | 陈悦 |
| **年龄** | 25 岁 |
| **公司/岗位** | 某传统软件公司，Java 后端开发（2 年经验） |
| **技术能力** | 熟练 Spring Boot、MyBatis、MySQL，对微服务和分布式了解不深 |
| **求职状态** | 准备跳槽到互联网中厂，目标 P6 级别 |
| **面试经验** | 2 年没面试过了，不太清楚现在面试的行情和重点 |
| **简历情况** | 有 2 段正式工作经历，2 个项目经验 |

#### 🎯 目标

- 了解当前互联网公司后端面试的重点和趋势
- 识别自己的技术短板（如微服务、分布式、系统设计）
- 通过充分准备，拿到 2-3 个满意的 offer

#### 😟 痛点

1. **技术栈落后** — 在传统公司用的技术相对老旧，和互联网公司要求有差距
2. **系统设计弱** — 没有设计高并发系统的经验
3. **面试行情不明** — 不知道现在面试重点考什么
4. **薪资没底** — 不知道自己值多少钱，不敢谈薪

#### 📱 使用场景

- 先在平台上选择"Java 高级开发 / P6"岗位，进行一次**困难模式**模拟面试
- AI 面试官考察了 JVM 调优、Redis 缓存、分布式事务等内容，发现很多答不上来
- 使用职业规划对比当前技能和目标岗位要求，识别出需要补足：微服务架构、Redis、消息队列、系统设计
- 查看市场洞察功能，了解当前 Java 后端的薪资范围和技能要求 Top 10

#### ✅ 成功标准

- 模拟面试评分从 65 提升到 85+
- 对微服务、Redis、消息队列等高频考点能够流畅回答
- 成功拿到 1 家互联网中厂的面试机会并进入终面

### P-05：职业迷茫期用户

> *"毕业半年了，做了两份工作都不喜欢，不知道自己到底适合做什么……"*

| 字段 | 内容 |
|------|------|
| **姓名** | 赵小棠 |
| **年龄** | 23 岁 |
| **背景** | 某二本院校，信息管理与信息系统专业 |
| **工作经历** | 毕业半年换了 2 份工作（行政 → 销售），都不喜欢 |
| **技能情况** | 会基础 Excel、PPT，大学学过一点 Python 和 SQL |
| **求职状态** | 离职状态，非常焦虑，不知道自己适合什么方向 |
| **面试经验** | 面试过行政岗和销售岗，但技术岗零经验 |

#### 🎯 目标

- 找到适合自己的职业方向
- 了解不同岗位需要什么技能
- 获得一个可行的转行路径

#### 😟 痛点

1. **自我认知不清** — 不知道自己的兴趣和优势在哪
2. **信息过载** — 网上职业建议太多，越看越乱
3. **缺乏行动力** — 知道应该学东西，但没人告诉具体学什么
4. **经济压力** — 不能脱产学习太久，需要快速找到方向

#### 📱 使用场景

- 注册后先填写个人信息，系统推荐了几个适合她背景的岗位方向
- 她选择了"数据分析师"和"产品经理"两个方向各进行一次轻松模式的模拟面试
- 对比两次面试体验和报告，发现自己对数据分析更感兴趣、回答也更流畅
- 使用职业规划制定了一份"零基础转行数据分析"的 3 个月学习计划
- 学习过程中，定期回来做提升测评，看到自己的进步

#### ✅ 成功标准

- 确定了一个清晰的职业方向（数据分析师）
- 按照学习计划坚持学习了 1 个月，能独立完成一个简单数据分析项目
- 投递了 5 家数据分析岗位并收到面试邀请

## 五、用户旅程简图

### 典型用户旅程（以 P-02 李思雨为例）

```
发现平台 → 注册登录 → 上传简历 → 选择"产品经理"岗位
    ↓                                              ↓
查看面试报告 ← 完成 8 轮模拟面试 ← AI 连续追问 ← 选择困难模式
    ↓
识别薄弱点（表达逻辑、项目深度）
    ↓
反复练习同一岗位面试（每天 1 次）
    ↓
第 10 次面试评分从 58 → 82
    ↓
在真实面试中成功通过二面 🎉
```

## 六、反用户画像（非目标用户）

以下人群不属于 Career-Copilot 的核心目标用户：

| 类型 | 原因 |
|------|------|
| **资深行业专家（10年+）** | 面试经验丰富，不需要 AI 模拟；职业路径已定型 |
| **HR / 招聘方** | 平台定位为求职者工具，非招聘管理平台 |
| **初中及以下学生** | 尚未面临求职需求，平台内容不适用 |
| **非求职目的的职场人士** | 如已在一家公司工作多年且无跳槽计划 |
