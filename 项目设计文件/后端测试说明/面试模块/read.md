# 面试模块 — Swagger UI 测试说明

## 接口前缀

所有接口：`/api/interviews/*`

## 认证方式

所有接口均需 **Bearer Token** 认证：

1. 先调用 `POST /api/auth/login` 获取 `accessToken`
2. 在 Swagger UI 右上角点击 **Authorize** 按钮
3. 输入 `Bearer <accessToken>`（例如 `Bearer eyJhbGciOi...`）

---

## 接口列表

### 1. 创建面试 — `POST /api/interviews`

| 项目 | 内容 |
|------|------|
| 功能 | 创建 AI 模拟面试，自动生成第一道题目 |
| 认证方式 | ✅ 需要 Bearer Token |
| 请求体参数 | `targetPosition`（必填，目标岗位）、`difficulty`（可选，难度）、`resumeId`（可选，关联简历 ID） |
| 请求体示例 | `{ "targetPosition": "前端开发工程师", "difficulty": "mid", "resumeId": "cm8abc123..." }` |
| 注意事项 | ⚠️ `difficulty` 可选值：`junior` / `mid` / `senior`，不传则默认为 `mid` |
| 测试建议 | 可以先上传简历获取简历 ID（或跳过简历直接填岗位名称和难度），创建成功后返回结果中包含面试 ID 和第一道面试题 |

### 2. 面试历史 — `GET /api/interviews`

| 项目 | 内容 |
|------|------|
| 功能 | 获取当前用户的面试记录列表（分页） |
| 认证方式 | ✅ 需要 Bearer Token |
| 查询参数 | `page`（可选，默认 1）、`limit`（可选，默认 10）、`status`（可选，筛选状态） |
| 示例 | `GET /api/interviews?status=in_progress` — 查看进行中的面试 |
| 注意事项 | ⚠️ `status` 可选值：`in_progress`（进行中）/ `completed`（已完成）/ `cancelled`（已取消） |
| 测试建议 | 创建面试后可用此接口查看面试列表及状态 |

### 3. 面试详情 — `GET /api/interviews/:id`

| 项目 | 内容 |
|------|------|
| 功能 | 获取面试的详细信息及完整对话记录 |
| 认证方式 | ✅ 需要 Bearer Token |
| 路径参数 | `id`（面试 ID） |
| 注意事项 | ⚠️ 只有该面试的创建者可以查看 |
| 测试建议 | 创建面试后从返回结果中复制 ID，传入此接口查看完整信息 |

### 4. 面试对话 — `GET /api/interviews/:id/messages`

| 项目 | 内容 |
|------|------|
| 功能 | 获取面试过程中的全部对话消息列表 |
| 认证方式 | ✅ 需要 Bearer Token |
| 路径参数 | `id`（面试 ID） |
| 注意事项 | ⚠️ 仅返回消息列表，不包含面试元数据；如需完整信息请使用面试详情接口 |
| 测试建议 | 提交几轮回答后调此接口查看对话记录 |

### 5. 提交回答 — `POST /api/interviews/:id/answer`

| 项目 | 内容 |
|------|------|
| 功能 | 提交用户对当前题目的回答，AI 评估后返回评分、反馈和下一题 |
| 认证方式 | ✅ 需要 Bearer Token |
| 路径参数 | `id`（面试 ID） |
| 请求体参数 | `content`（用户回答内容） |
| 请求体示例 | `{ "content": "useState 是 React 的基础 Hook，用于在函数组件中管理状态..." }` |
| 注意事项 | ⚠️ **关键**：`content` 字段中可包含换行和代码，若粘贴多段文字或代码块，Swagger UI 会保留换行符，后端已做兼容处理（自动转义控制字符） |
| 注意事项 | ⚠️ 提交后会返回 AI 评估结果，包含：`score`（评分 0-100）、`feedback`（评语）、`nextQuestion`（下一题，面试结束时此项为假值） |
| 注意事项 | ⚠️ 长回答（超过 500 字）可能触发较长的 AI 响应延迟（5-15 秒），请耐心等待 |
| 测试建议 | 1. 先创建面试获取第一道题<br>2. 提交回答，查看 AI 评分和反馈<br>3. 根据 `nextQuestion` 决定是否继续答题 |

### 6. 生成面试报告 — `POST /api/interviews/:id/feedback`

| 项目 | 内容 |
|------|------|
| 功能 | 根据面试对话记录生成多维度评分报告 |
| 认证方式 | ✅ 需要 Bearer Token |
| 路径参数 | `id`（面试 ID） |
| 注意事项 | ⚠️ 需要在完成至少一轮问答后调用，否则报告内容可能不完整 |
| 测试建议 | 完成多轮问答后调用此接口，查看 AI 生成的多维度评估报告 |

### 7. 结束面试 — `POST /api/interviews/:id/complete`

| 项目 | 内容 |
|------|------|
| 功能 | 手动结束面试，将状态标记为已完成 |
| 认证方式 | ✅ 需要 Bearer Token |
| 路径参数 | `id`（面试 ID） |
| 注意事项 | ⚠️ 面试完成后无法再提交回答；可在结束前先生成反馈报告 |
| 测试建议 | 完成全部问答后调用此接口结束面试，再调用生成报告接口获取最终评估 |

### 8. 删除面试 — `DELETE /api/interviews/:id`

| 项目 | 内容 |
|------|------|
| 功能 | 删除指定的面试记录及其所有对话消息（级联删除） |
| 认证方式 | ✅ 需要 Bearer Token |
| 路径参数 | `id`（面试 ID） |
| 注意事项 | ⚠️ 只能删除自己创建的面试记录；删除操作不可恢复 |
| 测试建议 | 1. 创建一个测试面试<br>2. 调用删除接口<br>3. 再次查询面试列表确认已删除 |

---

## 通用注意事项

1. **面试流程**：创建面试 → 查看题目 → 提交回答（反复） → 生成报告 → 结束面试
2. **控制字符**：提交回答时若回答含多行文本，后端已自动处理 `\n` 等未转义控制字符，无需手动转义
3. **AI 延迟**：每题 AI 评估耗时约 3-10 秒，请耐心等待响应
4. **超时配置**：AI 调用外部大模型 API（DeepSeek），超时时间已设为 60 秒
5. **ID 管理**：创建面试后请记录返回的面试 ID（`id` 字段），后续所有操作都需要此 ID
