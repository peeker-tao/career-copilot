## 一、引言

### 1.1 目的

本文档旨在全面定义 Career-Copilot 平台的功能需求和非功能需求，为系统设计、开发和测试提供依据。本文档的主要读者包括项目团队成员、指导老师及评审人员。

### 1.2 适用范围

本文档适用于 Career-Copilot 项目的 MVP 版本（V1.0），涵盖用户管理、简历管理、AI 模拟面试、职业规划四个核心模块。

### 1.3 术语表

| 术语 | 说明 |
|:----:|------|
| LLM | 大语言模型（Large Language Model） |
| JWT | JSON Web Token，用户认证令牌 |
| WebSocket | 全双工通信协议，用于实时对话 |
| Bull | 基于 Redis 的消息队列 |
| Prisma | Node.js ORM 框架 |
| MVP | 最小可行产品（Minimum Viable Product） |

### 3.2 详细功能需求

#### EPIC 1：用户账户管理

##### FR-001 用户注册

| 字段 | 内容 |
|:----:|------|
| **描述** | 访客通过邮箱和密码注册成为平台用户 |
| **输入** | 邮箱、密码（≥6位）、昵称（2-20字符） |
| **输出** | 用户信息 + Token 对（accessToken + refreshToken） |
| **前置条件** | 邮箱未被注册 |
| **后置条件** | 用户创建成功，返回 Token 自动登录 |
| **验收标准** | ① 邮箱格式校验 ② 密码长度 ≥ 6 ③ 昵称 2-20 字符 ④ 注册后自动登录 ⑤ 邮箱重复返回 409 错误 |
| **实现状态** | ✅ 后端已实现（`POST /api/auth/register`） |

##### FR-002 用户登录

| 字段 | 内容 |
|:----:|------|
| **描述** | 已注册用户通过邮箱和密码登录 |
| **输入** | 邮箱、密码 |
| **输出** | 用户信息 + 新 Token 对 |
| **前置条件** | 账号已注册且密码正确 |
| **后置条件** | 返回 JWT Token 对 |
| **验收标准** | ① 邮箱密码校验 ② 错误密码明确提示 ③ 返回 Token 对 |
| **实现状态** | ✅ 后端已实现（`POST /api/auth/login`） |

##### FR-003 Token 刷新

| 字段 | 内容 |
|:----:|------|
| **描述** | 使用 Refresh Token 轮换获取新的 Token 对 |
| **输入** | refreshToken |
| **输出** | 新的 accessToken + refreshToken |
| **前置条件** | refreshToken 有效 |
| **验收标准** | ① Token 轮换机制 ② 旧 Token 失效 |
| **实现状态** | ✅ 后端已实现（`POST /api/auth/refresh`） |

##### FR-004 个人资料管理

| 字段 | 内容 |
|:----:|------|
| **描述** | 用户查看和修改个人资料 |
| **输入** | 昵称、头像 URL、教育背景、目标岗位 |
| **输出** | 更新后的用户信息 |
| **前置条件** | JWT 认证通过 |
| **接口** | `GET /api/auth/profile`、`PATCH /api/auth/profile` |
| **实现状态** | ✅ 后端已实现 |

#### EPIC 2：简历管理

##### FR-005 简历上传

| 字段 | 内容 |
|:----:|------|
| **描述** | 用户上传 PDF/Word 格式简历文件 |
| **输入** | 文件（.pdf / .docx）、简历标题 |
| **输出** | 简历基本信息 + 上传状态 |

| 字段 | 内容 |

| **提取内容** | 技能标签、工作经历、教育背景、项目经验 |
| **处理流程** | 文本提取 → LLM 结构化 → 持久化 |
| **回退策略** | PDF 文本失败 → OCR 识别 → LLM 解析 |
| **实现状态** | ✅ 后端已实现（`ResumeParser` + OCR 回退） |

##### FR-007 简历 CRUD

|:----:|------|
| **描述** | 用户管理自己的多份简历 |
| **接口** | `GET /api/resumes`（列表）、`GET /api/resumes/:id`（详情）、`PATCH /api/resumes/:id`（更新）、`DELETE /api/resumes/:id`（删除） |
| **实现状态** | ✅ 后端已实现 |

#### EPIC 3：AI 模拟面试（核心功能 ⭐）

##### FR-008 面试准备

| 字段 | 内容 |
|:----:|------|
| **描述** | 用户选择目标岗位和难度创建面试 |
| **输入** | 目标岗位、难度（easy/medium/hard）、关联简历（可选） |
| **输出** | 面试会话 + 第一个面试题 |
| **实现状态** | ✅ 后端已实现（`POST /api/interviews`） |

##### FR-009 AI 面试对话

| 字段 | 内容 |
|:----:|------|
| **描述** | 用户与 AI 面试官进行多轮文字对话 |
| **流程** | AI 出题 → 用户回答 → AI 评分反馈 → 追问/换题 → ... → 结束 |
| **对话轮次** | 5-8 轮 |
| **评分机制** | 1-5 分制，含即时反馈 |
| **传输方式** | HTTP（`POST /api/interviews/:id/answer`）+ WebSocket（流式） |
| **实现状态** | ✅ 后端已实现（`AiInterviewService` + `InterviewGateway`） |

##### FR-010 面试报告

| 字段 | 内容 |
|:----:|------|
| **描述** | 面试结束后生成综合评价报告 |
| **报告内容** | 综合评分、维度评分（专业技能/沟通表达/逻辑思维/项目经验）、优缺点、改进建议 |
| **实现状态** | ✅ 后端已实现（`POST /api/interviews/:id/feedback`） |

##### FR-011 面试历史

| 字段 | 内容 |
|:----:|------|
| **描述** | 查看历史面试记录 |
| **筛选条件** | 岗位、难度、状态、时间范围 |
| **输出** | 面试列表（分页）+ 评分趋势 |
| **实现状态** | ✅ 后端已实现（`GET /api/interviews`） |

| NFR-001 | 普通 API 响应时间 | ≤ 2 秒（不含 LLM 调用） |
| NFR-002 | LLM 接口响应时间 | ≤ 30 秒（含流式输出） |
| NFR-003 | 并发用户支持 | ≥ 100 同时在线 |

### 4.2 安全需求

| 需求编号 | 描述 | 实现方式 |
|:--------:|------|:--------:|

| NFR-007 | 文件上传安全 | 类型校验 + 大小限制 + 角色守卫 |
| NFR-008 | 参数校验 | class-validator DTO 校验 |

### 4.3 可用性需求

## 五、数据需求

### 5.1 数据实体

| 实体 | 说明 | 核心字段 |
|:----:|:----:|----------|
| User | 用户账号 | email, passwordHash, name, role |
| Resume | 简历信息 | title, fileUrl, parsedData, skills |
| Interview | 面试会话 | targetPosition, difficulty, status, score |
| InterviewMessage | 对话消息 | role, content, questionType, feedback |
| CareerPlan | 职业规划 | targetPosition, currentSkills, gapSkills, roadmap |

### 5.2 数据约束

- 邮箱全局唯一
- 简历关联所属用户
- 面试关联用户和简历（可选）
- 消息关联所属面试会话
