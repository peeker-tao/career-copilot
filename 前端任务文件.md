## 二、API 层封装（基础架构）

### 2.1 Axios 实例与拦截器 — `src/api/client.ts`

**职责：** 创建统一的 Axios 实例，配置请求/响应拦截器。

**实现要点：**

1. 创建 Axios 实例，`baseURL` 从环境变量读取（`VITE_API_BASE_URL`，默认 `http://localhost:3001/api/v1`）
2. **请求拦截器：** 从 localStorage 读取 accessToken，自动附加 `Authorization: Bearer <token>` 头
3. **响应拦截器：**
   - 成功时直接返回 `response.data`
   - 401 时自动尝试刷新 Token（调用 `/auth/refresh`）
   - 刷新成功后重放原始请求
   - 刷新失败则清除 Token 并跳转登录页
4. 统一错误处理，将后端错误码映射为中文提示（如 `'401' → '登录已过期，请重新登录'`）

```typescript
// 伪代码结构
const client = axios.create({ baseURL: import.meta.env.VITE_API_BASE_URL });

client.interceptors.request.use((config) => {
  const token = getAccessToken();
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

client.interceptors.response.use(
  (response) => response.data,
  async (error) => {
    if (error.response?.status === 401) {
      // 尝试刷新 token
      const refreshed = await refreshToken();
      if (refreshed) return client.request(error.config); // 重放
      // 刷新失败，跳转登录
    }
    return Promise.reject(error);
  }
);
```

按模块拆分到不同的文件，每个函数返回类型化的 Promise。例如 `src/api/auth.ts`：

### 3.2 认证状态管理 — `src/store/authStore.ts`

```typescript
interface AuthState {
  user: UserProfile | null;
  isAuthenticated: boolean;
  setAuth: (user: UserProfile, accessToken: string, refreshToken: string) => void;
  logout: () => void;
  updateUser: (data: Partial<UserProfile>) => void;
}
```

**实现要点：**

- 应用启动时从 localStorage 读取 Token，调用 `GET /auth/profile` 验证有效性
- `logout()` 清除 localStorage 中的 Token，重置 user 为 null，跳转 `/login`
- 使用 `persist` 中间件将 user 信息持久化到 localStorage

## 十一、开发注意事项

### 11.1 样式约定

- 全局使用 TailwindCSS 工具类处理间距、颜色、排版
- Ant Design 组件使用其自带样式，通过 `ConfigProvider` 设置主题色（`#1890ff`）
- 面试对话等复杂交互页面使用 CSS Modules 隔离样式
- 颜色变量统一在 `tailwind.config.js` 的 `theme.extend.colors` 中定义

### 11.2 状态管理原则

- **Zustand** 管理全局状态（认证信息、面试会话）
- **React Query / SWR**（可选）管理服务端状态缓存
- 页面内临时状态使用 React 本地 `useState`
- URL 参数使用 React Router 的 `useParams` / `useSearchParams`

### 11.3 错误处理原则

- 所有 API 调用使用 `try-catch` 包裹
- API 层拦截器统一处理 401 和网络错误
- 页面级错误使用 Ant Design `message.error()` 提示
- 不可恢复的错误显示 ErrorBoundary 降级 UI

### 11.4 性能注意事项

- 简历列表、面试历史列表数据量大时分页加载
- ECharts 图表组件使用 `lazy` 加载
- WebSocket 连接在页面卸载时及时关闭
- 面试消息列表渲染使用虚拟滚动（react-window，可选）
